React
리엑트는 페이스북사에서 만든 자바스크립트 라이브러리입니다.
페이스북에서 자체적으로 이용되고 있을 뿐만 아니라 많은 웹 어플리케이션이 리엑트로 만들어졌습니다.

Angular.js(프레임워크), vue.js(프레임워크와 라이브러리 중간 단계)
* 프레임워크
  집, 건축물, 미리 만들어진 틀이 존재
* 라이브러리
  도구, 인테리어 제품

"리엑트는 유저 인터페이스를 만들 수 있는 라이브러리입니다"
-> 사용자에게 보여지는 UI를 만들고 사용자가 클릭을 하거나 이벤트가 발생하게 되면 그 이벤트에 맞게 반응하도록 만들어진 라이브러리입니다.


복잡한 웹 어플리케이션을 만들때 MVC패턴을 사용합니다.
                    
                         MVC 패턴
                       어플리케이션
        Model              View           Controller
    데이터베이스관련      사용자화면       서버쪽, 로직

"리엑트는 이 중 view를 담당"
-> mvc패턴으로 개발하는 방식으로 개발할때 react는 view만 담당한다. 즉, 리엑트는 어떠한 서버언어와 같이 개발이 가능하다
( 이와 다르게 화면에 데이터를 뿌려주고 프로그래밍하는거를 php로 개발
  -> php는 화면단 서버단 모두 담당
  -> 서버랜더링방식으로 서버쪽에서 화면을 만들어 뿌려주는 방식 )


React: 웹 어플리케이션 UI개발
React Native: 모바일 어플리케이션 개발

버튼 하나와 같이 한 가지 기능을 수행하는 ui단위를 말한다. 서로 독립적이고 재사용이 가능한 것을 의미합니다.

                                   root
                    header        content        footer
                 logo    menu     article ..   

사이트를 트리로 표현했을때 각 요소단위를 독립적인 컴포넌트로 만들어서 조립해 놓은 것을 리엑트 어플리케이션이라고 합니다

컴포넌트 코드
컴포넌트는 클래스 혹은 함수로 만드는 방법 2가지이다.
import React from 'react';  //리엑트 문법 사용
class MenuButton extends Component {  //Component 상속받음
    state = {                     
        num: 0
    };

    render(){
        return <button>
        {this.state.num}
        </button>;
    }
}
export default MenuButton;

state: 데이터 상태를 나타내는 객체
render(): 사용자에게 현재 컴포넌트를 화면에 어떻게 보여줄지 나타내는 함수
          함수 내에서는 JSX문법으로 표기
* JSX 문법
페이스북에서 만든 화면을 표현하는 문법이며 랜더링하면 자바스크립트로 변환
- 태그는 꼭 닫아야 한다
- 두개이상의 태그는 무조건 하나의 태그로 감싸야한다
- 주석은 {/* */}
- css속성은 스네이크 표기법 -> 카멜표기법 (backrround-color -> backgroundColor)
- 여러 태그를 감쌀때 불필요한 div를 사용하고 싶지 않다면 <></>를 사용하여 감쌀 수 있다


"리엑트는 데이터가 변경될때마다 변경컴포넌트 밑으로 하위컴포넌트까지 어플리케이션 전체를 다시 랜더링한다"
개발자는 데이터가 변경이 되면 해당 부분을 업데이트 해야하는 것을 걱정하지 않아도 된다
리엑트는 state에 맞게 render()함수에서 어떻게 표현될 것인지 한번만 정의하면 알아서 render함수를 다시 호출해주기 때문입니다.
-> 메모리에 현재 화면에 대한 가상 돔트리를 만드는데 항상 이전 화면에 대한 가상 돔트리를 기억하고 있다.
   따라서 통째로 갱신하긴 하지만 이전 가상 돔트리와 비교하여 변경사항이 없으면 넘어가고 변경사항이 있는 부분만 갱신한다

create react-app
페이스북에서 만든 리엑트를 개발하는 개발자들이 공통적으로 사용하는 유용한 툴을 한번에 자동으로 설치할 수 있도록 도와주는 툴입니다
yarn create react-app 프로젝트명

프로젝트 실행
cd 프로젝트명
yarn start

프로젝트 삭제
rm -rf 프로젝트명


리엑트 프로젝트 구성
yarn.lock: package.json의 부가적인 것들
README.md: 깃에 올릴때 프로젝트에 대한 설명
package.json: npm에서 버전관리할때 프로젝트에서 외부적으로 쓰고 있는 라이브러리의 버전들과 스크립트들이 명시되어 있습니다
              yarn eject명령 실행시 숨겨진 모듈확인 가능
.gitignore: 여기에 명시되어 있는 파일들은 깃에 저장되지 않습니다
public 폴더: 사용자에게 배포할때 외부적으로 보여지는 이미지, html 등 리소스를 포함하는 디렉토리
src: 소스코드를 포함하는 디렉토리

숨겨진 툴
create react-app을 실행하면서 자동으로 깔림
BABEL
최근 문법을 해석하지 못하는 예전 버전 브라우저에서 babel을 설치하여 최근 문법을 과거 문법으로 변환 
자바스크립트 트랜스컴파일러입니다. ECMA Script 2015버전 코드를 이전 버전으로 변환해주는 툴입니다. 
TypeScript(변수타입이 없는 자바스크립트와 달리 타입을 결정하는 스크립트), JSX와 같은 순수 자바스크립트가 아닌 문법을 브라우저가 이해할 수 있도록 자바스크립트 코드로 변환해주는 역할을 하기도 합니다

Webpack
소스코드나 리소스 이미지들을 한번에 번들단위로 사용자에게 제공할수 있도록 도와주는 툴입니다
또한 제공할때 자바스크립트 코드를 줄여주고 긴 변수나 함수의 이름을 해커들이 알아볼 수 없도록 변경해주는 역할도 합니다.

Jest

함수단위로 테스트가 가능하다

PostCSS
CSS전처리기 중 하나입니다
정해진 프레임워크에 따라서 css를 작성하게 되면 그 css를 브라우저가 이해할 수 있는 css코드로 변환합니다. sass나 less(css로 프로그래밍)와 비슷하지만 PostCSS는 다양한 플러그인이 있어서 확장성이 좋습니다
* sass less
css를 보다 편리하게 작성하기 위한 추가문법을 제공하고 작성된 코드를 css로 변환하는 기능도 제공한다
