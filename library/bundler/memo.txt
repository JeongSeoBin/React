# bundler
- parcel, webpack, rollup, ...
- 역할
1. 애플리케이션에 필요한 모든 파일들을 모듈 단위로 나누어 최소한의 파일묶음(번들러) 생성
   -> 네트워크 병목현상 해결 (파일 개수를 줄이므로)
   -> 모듈 단위 코딩이 가능해져 코드 가독성과 유지보수 효율을 높일 수 있다 (모듈단위로 ㅈ코드를 작성하여도 하나의 파일로 만들어주니까)
2. 서드 파티 기능 제공
   - uglyfy
     js파일을 외부에서 알아 보기 힘들게 코드를 변환
   - transpile
     web에서 동작하는 언어는 html, css, js이다
     하지만 실제 코드를 작성할때에는 react, sass, ts, vue, ... 으로 작성한다
     이들은 웹에서 동작할 수 없기 떄문에 변환하는 과정이 필요하다
     이 역할을 bundler가 수행한다
     bundler가 문법을 이해하고 직접 변환하는 것이 아니라 실제 변환은 외부 패키지의 도움을 받아서 변환한다
   - babel
     최신 자바스크립트 문법을 es5문법으로 변환
   
# parcel bundler vs webpack bundler
parcel
- 구성없는 단순한 자동 번들링
- 소/중형 프로젝트에 적합
webpack
- 꼼꼼한 구성
- 중/대형 프로젝트에 적합
